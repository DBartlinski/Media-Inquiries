What is the Media Inquiries Dashboard?

A collaborative tool for managing media inquiries from start to finish

Purpose

  • Replace Microsoft Planner with a lightweight, shareable dashboard
  • Track media inquiries through workflow stages (Not Started → In Progress → Completed)
  • Organize by priority (Urgent, Important, Medium, Low)
  • Archive completed inquiries for historical reference
  • Import existing data from CSV exports
  • Search and filter inquiries in real-time

Key Features

  • Kanban Board: Visual workflow with drag-and-drop columns
  • Shared Credentials: All team members use one token
  • Real-Time Sync: Changes save to GitHub instantly
  • No Backend Needed: Data stored securely in GitHub
  • Offline Support: Browser caching if GitHub is unavailable
  • Smart CSV Import: Merge new data with existing tasks

Real-World Example

How the Media Inquiries Dashboard is used by VHA Communications

📰 Scenario: A News Organization Submits an Inquiry

Monday, 9:00 AM: USA TODAY sends an urgent inquiry about VHA's new vaccine initiative. Jane from VHA Communications receives the email.

Action: Jane opens the Dashboard and clicks "New Inquiry". She fills in:

Title: USA TODAY - VHA Vaccine Initiative Q&A Priority: Urgent Status: In Progress Assigned To: Jane; Dr. Smith (Subject Matter Expert) Due Date: Tuesday, 5:00 PM Description: Questions about vaccine distribution, safety protocols, and timeline Checklist: ☐ Received request from USA TODAY ☐ Received specific questions ☐ Looped in Dr. Smith (SME) ☐ Drafted responses ☐ Final review and approval

Monday, 2:00 PM: Dr. Smith logs into the Dashboard (same credentials, same URL). He sees Jane's inquiry on the board under "In Progress" column. He clicks the card and adds his responses to the Description.

Tuesday, 10:00 AM: Jane refreshes the board and sees Dr. Smith's updates. She checks off the "Looped in SME" and "Drafted responses" items. The checklist progress bar updates instantly.

Tuesday, 4:00 PM: Both Jane and Dr. Smith review the response. Jane updates the task status to "Completed" and moves the card. The dashboard automatically saves all changes to GitHub.

Result: The inquiry is tracked from start to finish. The entire team can see what's pending, in progress, or completed. When a new inquiry comes in, Jane can quickly search for similar past inquiries to avoid duplication of effort.

Frequently Asked Questions

What happens if GitHub goes down? ▼
The dashboard caches data in your browser's local storage. You can continue using the app offline, and all changes will sync to GitHub automatically when it's back online.
Is my data secure? ▼
Your data is stored in a private GitHub repository and transmitted over HTTPS. The Personal Access Token is stored only in your browser's localStorage and is never sent to any external servers. GitHub is SOC 2 Type II certified.
What if someone leaves the team? ▼
If a team member leaves, you can rotate the shared Personal Access Token by generating a new one in GitHub settings (Settings → Developer settings → Personal access tokens). Then share the new token with remaining team members.
Can we see who made what changes? ▼
With the shared token approach, all changes are attributed to the GitHub account owner (DBartlinski). If you need individual audit trails, consider using fine-grained GitHub tokens instead (each team member gets their own), but this requires more setup.
How often does data sync? ▼
Changes sync to GitHub instantly when you save. However, other team members won't see updates automatically — they need to refresh their browser (F5) to see your changes. The dashboard will notify you when someone else has synced new data.
Can we customize the checklist items? ▼
Yes! When you create a new inquiry, the checklist is pre-filled with default items (Received request, received questions, looped in SME, etc.), but you can add or remove items as needed for each task.
What about recurring inquiries? ▼
You can mark inquiries as "Is Recurring" in the form. This is a tag for your reference. The dashboard doesn't automatically create recurring instances — you can manually import updated CSVs or create new cards as needed.
Can we export the data? ▼
The data is stored as `data.json` in your GitHub repository, so you can download it anytime via GitHub's web interface or clone the repo. You can also use the GitHub API to extract data programmatically.
What if I see "Repository not found" error? ▼
Check these:
  • Owner: DBartlinski (capital D and B)
  • Repository: Media-Inquiries (with hyphen, not space)
  • Token has repo scope permission
Can I use this for other types of tracking? ▼
Absolutely! While built for media inquiries, the dashboard can be used for any workflow that fits a Kanban model: bug tracking, FOIA requests, research tasks, event planning, or any project that needs priority-based task tracking.
